# Chat 交互测试路线（核心流+异常流）

## 测试路线用例表

| 编号 | 路线 | 前置条件 | 操作步骤 | 期望结果 |
|---|---|---|---|---|
| A1 | 入口导航 | 服务已启动 | 打开 `/frontend/`，点击 `AI 问答` | 跳转到 `/frontend/chat/` |
| A2 | 返回搜索 | 在 `/frontend/chat/` | 点击 `data-testid=back-to-search-btn` | 返回 `/frontend/` 且页面可继续检索 |
| B1 | 新建会话 | Chat 页可用 | 点击 `data-testid=new-session-btn` | 会话列表新增项，激活状态正确 |
| B2 | 会话持久化 | 已创建会话 | 刷新页面 | 会话列表仍可见，详情可加载 |
| B3 | 切换会话 | 至少两个会话 | 点击不同 `data-testid=session-item` | 消息区切换到对应历史 |
| C1 | 空消息拦截 | 会话已选中 | 输入空格并发送 | 前端拦截，不调用提交接口 |
| C2 | 发送消息 | 会话已选中 | 输入文本并发送 | 出现用户气泡与助手运行态 |
| C3 | SSE 正常链路 | 后端可推流 | 观察 SSE 事件 | 顺序含 `started -> delta -> completed` |
| C4 | 引用渲染 | 快速模式回答含 citations | 完成后查看回答 | 出现引用区块 `data-testid=message-citations` |
| D1 | 快速模式 | mode=fast | 提问并等待完成 | 返回 answer + citations/retrieval 信息 |
| D2 | Agent 模式 | mode=agent | 提问并等待完成 | 可看到 tool_call 相关轨迹，最终结束 |
| E1 | provider 校验 | 会话已存在 | provider 传非法值 | 返回 400 `invalid_provider`，前端提示错误 |
| E2 | session 校验 | 无效会话 id | 请求详情/提交消息 | 返回 404 `session_not_found` |
| E3 | SSE 失败链路 | 注入失败响应 | 触发失败流 | 顺序含 `started -> failed`，前端显示失败态 |
| F1 | 图谱回归 | 服务已启动 | 调用 `/graph/search` | 返回 200，原功能不回归 |
| F2 | 文献回归 | 文献服务可用 | 调用 `/literature/search` `/literature/answer` | 返回 200 或受控错误，不出现接口断裂 |

## 缺陷分级

- P0（阻断）：无法进入 Chat、无法发送消息、SSE 无终态、服务崩溃。
- P1（高优）：核心链路可用但结果错误（错误会话、错误模式、参数校验失效）。
- P2（中优）：交互体验缺陷（状态不同步、文案异常、引用渲染缺失）。
- P3（低优）：样式轻微问题、非关键提示语偏差。

## 循环退出条件

- 所有测试路线用例通过。
- P0/P1 缺陷数为 0。
- 关键路径 `搜索 -> Chat -> 提问 -> 流式 -> 引用 -> 会话切换` 连续两轮通过。
